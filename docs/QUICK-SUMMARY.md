# 재사용 가능한 제안서 시스템 - 요약

## 목적

**MD 파일 하나로 의료기관 제안서를 빠르게 생성**

---

## 핵심 구조

```
┌────────────────────────────────────────────────────────┐
│                    content/proposal.md                  │
│                    (콘텐츠 작성)                        │
└─────────────────────────┬──────────────────────────────┘
                          │ npm run generate
                          ▼
┌────────────────────────────────────────────────────────┐
│                    scripts/parse-md.ts                  │
│                    (파서 + 검증)                        │
└─────────────────────────┬──────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────┐
│               src/data/proposal-data.ts                 │
│               (자동 생성된 TypeScript)                  │
└─────────────────────────┬──────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────┐
│                   src/components/                       │
│                   (슬라이드 렌더링)                     │
└────────────────────────────────────────────────────────┘
```

---

## 설정 파일 (config/)

| 파일 | 역할 | 수정 빈도 |
|------|------|----------|
| `theme.ts` | 색상, 패턴, 타이포그래피 | 프로젝트당 1회 |
| `structure.ts` | 6막 구조 정의 | 거의 없음 |
| `meta.ts` | 의료기관/회사 정보 | 프로젝트당 1회 |

---

## 새 프로젝트 생성 워크플로우

```bash
# 1. 템플릿 복제
npx degit howon/automed-proposal 새의료기관-pro

# 2. 설정 수정
cd 새의료기관-pro
# config/theme.ts - 브랜드 색상
# config/meta.ts - 의료기관 정보

# 3. 콘텐츠 작성
# content/proposal.md

# 4. 데이터 생성
npm run generate

# 5. 개발 서버
npm run dev
```
